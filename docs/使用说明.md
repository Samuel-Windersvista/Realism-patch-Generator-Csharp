# SPT现实主义数值范围编辑生成器 v1.22 使用说明

本文档对应当前 C# 版项目的实际状态，覆盖 GUI、规则编辑、例外物品管理和审计工作流。

## 1. 版本定位

- 版本号：v1.22
- 运行形态：独立的 .NET GUI 工具
- 当前重点：规则编辑、补丁生成、审计检查、例外物品覆盖全部在同一工程内完成

## 2. 目录约定

程序默认以仓库根目录为数据根，依赖以下目录：

- input
- 现实主义物品模板
- output
- audit_reports
- rules

其中：

- rules 保存四大类规则，以及 rules/item_exceptions.json
- output 保存生成后的补丁结果
- audit_reports 保存审计报告

## 3. GUI 工作流

启动 GUI：

```powershell
dotnet run --project RealismPatchGenerator.Gui
```

主界面结构：

- 顶部：保存全部、重新加载、生成补丁、检查未遵循规则物品、语言切换、搜索
- 输出路径右侧：可选 Seed 输入框，留空表示每次生成重新采样，填写固定无符号整数可复现结果；支持一键清空，并可回填最近一次实际使用的 seed
- 左侧：规则分类树
- 中部：规则范围编辑表
- 底部：字段说明、例外总览、运行日志
- 额外入口：“例外物品”窗口用于处理具体 ItemID 的最终字段覆盖

常规使用顺序：

1. 重新加载规则，确认当前磁盘状态。
2. 在左侧选择分类，在中部调整数值范围。
3. 点击保存全部，把规则写回 rules 目录。
4. 点击生成补丁，把结果输出到 output。
5. 点击检查未遵循规则物品，生成审计报告并回看异常项。
6. 如有少数物品必须脱离规则单独处理，再进入“例外物品”窗口补充覆盖。

## 4. 例外物品管理

例外物品用于处理“某个具体 ItemID 必须保留或强制写入指定字段”的场景。

当前窗口流程：

1. 第一步按 Name 只搜索 output 中已经生成出来的物品。
2. 载入搜索结果后，右侧自动显示该物品当前顶层字段。
3. 在同一编辑区中选择字段、修改字段值，或新增当前物品所属大类允许的字段。
4. 点击“新增/修改字段”将当前编辑器中的字段和值写回字段列表。
5. 点击“保存物品”把当前 ItemID 的例外配置写入 rules/item_exceptions.json。

当前行为约束：

- 新增字段只允许来自当前物品所属大类的字段池
- 建议值范围来自当前最新规则数据，不是写死常量
- 数值在保存时会经过归一化和安全夹紧，避免极端越界
- 输出结构会尽量与“现实主义物品模板”中的对应大类格式保持一致

## 5. 输出与审计

当前生成行为：

- 结果写入 output
- 不会先清空整个输出目录
- 只覆盖本次需要写出的目标文件
- 每次生成都会在规则范围内重新采样，因此同一物品在多次生成之间可能出现不同数值
- 输出顺序会保留输入源文件中的条目顺序，方便用户按原始来源人工核对
- 霰弹当前按 12g、20g、23x75 三档分别套用弹药基础范围与武器口径补修

当前审计行为：

- 可审计武器、附件、弹药、装备四类结果
- consumable 和普通 cosmetic 不作为规则审计主范围
- attachment 中的 mod_profile_unresolved 不计入“未能校验附件范围”噪音
- 已在 item_exceptions 中明确覆盖的字段会被定向豁免，不会整件物品跳过检查

## 6. 启动方式

当前版本已移除独立 CLI 入口，生成与审计统一通过 GUI 执行。

开发环境下启动 GUI：

```powershell
dotnet run --project RealismPatchGenerator.Gui
```

如果需要复现某一轮生成结果：

- 在 GUI 的 Seed 输入框中填写固定无符号整数
- 点击生成补丁执行固定 seed 生成
- 需要回到随机模式时清空 Seed 输入框

## 7. 当前边界

- GUI 仍以中文版体验优先
