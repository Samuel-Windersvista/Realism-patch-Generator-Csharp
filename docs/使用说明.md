# SPT现实主义数值范围编辑生成器 v0.9 使用说明

本文档面向当前 C# 版独立程序的实际使用者，重点说明 GUI 的当前工作流，以及 CLI 的基础调用方式。

## 1. 当前版本定位

- 版本号：v0.9
- 当前重点：提供可用的结构化规则范围编辑 GUI，并保留 CLI 生成与检查能力
- 当前界面重点：中文版优先

## 2. 数据目录要求

程序默认在工程根目录下工作，依赖以下目录：

- input
- 现实主义物品模板
- output
- audit_reports
- rules

如果你在 GUI 中手动选择目录，该目录也需要同时包含以上核心结构，尤其是：

- input
- 现实主义物品模板
- rules

## 3. GUI 使用方式

启动 GUI：

```powershell
dotnet run --project RealismPatchGenerator.Gui
```

### 3.1 主界面结构

- 顶部：数据目录、保存全部、重新加载、生成补丁、检查未遵循规则物品、语言、搜索
- 左侧：规则分类树
- 中部：规则范围编辑表
- 底部：字段说明 / 运行日志

### 3.2 典型工作流

1. 启动 GUI 并确认数据目录正确。
2. 在左侧选择一个规则分类。
3. 选择具体 profile 或直接查看整个分类。
4. 在中部表格中调整最小值和最大值。
5. 观察底部字段说明，确认该字段的实际影响。
6. 点击保存全部，将修改写回 rules 目录中的规则文件。
7. 点击生成补丁，输出新的结果到 output。
8. 点击检查未遵循规则物品，确认 output 中是否存在未按规则落盘的条目。

### 3.3 编辑行为说明

- 表格当前只允许编辑数值范围，不暴露原始 JSON 文本。
- 最小值不能大于最大值。
- 已修改但未保存的条目会有明显视觉提示。
- 保存时程序会尝试对规则文件进行规范化写回。

### 3.4 搜索行为说明

搜索框当前会根据以下信息做过滤：

- profile 名称
- 字段原始键名
- 字段中英文显示名称
- 分类中英文名称

## 4. CLI 使用方式

启动生成：

```powershell
dotnet run --project RealismPatchGenerator.Cli
```

执行未遵循规则物品检查：

```powershell
dotnet run --project RealismPatchGenerator.Cli -- audit
```

常用可选参数：

- --output-dir：自定义待检查输出目录
- --report-file：自定义报告输出路径
- --include-ok：将正常项也写入报告
- --include-template-exports：将 output 下所有 json 一起纳入检查
- --fail-on-violations：发现违规字段时返回非 0 退出码

## 5. 当前版本限制

- 当前 GUI 以中文版体验为优先，英文版尚未继续细修。
- 规则中文名称映射已补充一批常用 profile，但仍可继续按项目术语微调。
- 当前自动化测试仍以核心与检查逻辑为主，GUI 交互自动化尚未补齐。

## 6. 建议使用顺序

建议每次改规则时遵循以下顺序：

1. 重新加载规则，确认当前磁盘状态。
2. 修改少量范围值。
3. 保存全部。
4. 生成补丁。
5. 检查未遵循规则物品。
6. 再根据检查结果继续微调。
